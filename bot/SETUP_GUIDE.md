# 🎪 Дискуссионный клуб по гидролатам - Бот

Полнофункциональный Telegram-бот для управления закрытым платным клубом с системой анкетирования, модерации и платежей.

## 📋 Функциональность

### Для пользователей
- ✅ Приветственное сообщение с описанием клуба
- 📝 Прохождение анкеты (вопрос-ответ формат)
- 📊 Отслеживание статуса анкеты (ожидание, одобрена, отклонена, оплачена)
- 💳 Получение реквизитов для оплаты после одобрения
- 🎉 Получение доступа к клубу после подтверждения платежа

### Для администраторов
- 📋 Просмотр анкет на проверку
- ✅ Одобрение анкет
- ❌ Отклонение анкет с указанием причины
- 💳 Подтверждение платежей и создание подписок
- 📊 Просмотр истории всех действий

### Для супер-администраторов
- 🎓 Управление анкетой (добавление/удаление/редактирование вопросов)
- 💰 Управление тарифными планами
- 🎟️ Создание промокодов (коллективные и индивидуальные скидки)
- 👥 Управление администраторами (добавление/удаление)
- 🔑 Всё что доступно обычным администраторам

## 🚀 Установка и запуск

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения
```bash
# Создайте .env файл или установите переменные окружения
export BOT_TOKEN="ваш_токен_бота"
export DATABASE_URL="postgresql+asyncpg://user:password@localhost/gidrolat_db"
```

### 3. Создание БД и таблиц
```bash
# БД создастся автоматически при запуске бота
# Или запустите миграции вручную:
alembic upgrade head
```

### 4. Инициализация данных

**Создание первого супер-администратора:**
```bash
python -m src.db_components.init_admin <ВАШ_TELEGRAM_ID> true
```

Пример:
```bash
python -m src.db_components.init_admin 123456789 true
```

**Создание стандартных тарифных планов:**
```bash
python -m src.db_components.init_payment_plans
```

Это создаст три плана:
- Месячный: 990₽ (30 дней)
- Полугодовой: 4990₽ (180 дней)
- Годовой: 8990₽ (365 дней)

### 5. Запуск бота
```bash
python main.py
```

## 📲 Статусы анкеты

| Статус | Описание | Переход |
|--------|---------|---------|
| `pending_review` | Ожидает проверки администратором | → approved / rejected |
| `approved` | Одобрена, отправлены реквизиты | → pending_payment |
| `rejected` | Отклонена | ✗ (можно переподать позже) |
| `pending_payment` | Ожидается оплата | → paid |
| `paid` | Оплачена, пользователь получил доступ | ✅ Доступ активен |

## 💰 Система платежей

### Процесс платежа:
1. Пользователь заполняет анкету
2. Админ одобряет/отклоняет анкету
3. Пользователю отправляются реквизиты для оплаты
4. Пользователь платит по реквизитам
5. Админ подтверждает платёж в боте
6. Создаётся подписка, пользователь получает доступ

### Скидки:
- **Коллективные промокоды**: Используются несколькими пользователями с ограничением на количество применений
- **Индивидуальные скидки**: Привязаны к конкретному пользователю

## 📝 Типы вопросов в анкете

1. **yes_no** - Вопрос с ответами "Да"/"Нет"
2. **text** - Развёрнутый текстовый ответ
3. **multiple_choice** - Выбор одного варианта из нескольких

## 🗄️ Структура БД

### Основные таблицы:

**users** - Пользователи
- id (Telegram ID)
- username
- first_name
- last_name

**survey_submissions** - Анкеты пользователей
- id
- user_id (FK)
- status (pending_review / approved / rejected / pending_payment / paid)
- answers (JSON)
- reviewer_id (ID админа, проверившего)
- reviewer_comment

**payment_plans** - Тарифные планы
- id
- name
- duration_days
- price
- is_active

**subscriptions** - Подписки пользователей
- id
- user_id (FK, уникальна)
- plan_id (FK)
- status (active / expired / cancelled)
- start_date
- end_date
- price_paid

**promo_codes** - Промокоды
- id
- code (уникален)
- discount_percent
- is_collective
- assigned_user_id (для индивидуальных)
- max_uses
- current_uses

**admin_users** - Администраторы
- id (Telegram ID)
- username
- first_name
- is_super_admin
