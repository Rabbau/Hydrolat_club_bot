# Telegram Bot (Gidrolat)

Бот для приема анкет, модерации, оплаты подписок и управления контентом сообщений через админ-панель.

## Текущие возможности

### Пользователь
- `/start` — регистрация пользователя и вход в меню.
- Заполнение анкеты (вопросы берутся из БД).
- Кнопка `Статус профиля`:
  - статус анкеты,
  - персональная скидка,
  - скидка по промокоду,
  - статус активной подписки.
- Кнопка `Промокод` — применение промокода к последней анкете.
- Кнопка `Тарифы` — просмотр всех активных тарифных планов.
- Скрытая команда `/id` — показать Telegram ID пользователя.

### Админ
- Меню `Модерация`:
  - просмотр анкет на проверке,
  - детальный просмотр ответов,
  - одобрение анкеты,
  - одобрение анкеты с персональной скидкой,
  - отклонение анкеты.
- Раздел `Ожидают оплату`:
  - выбор анкеты,
  - подтверждение оплаты,
  - выбор тарифного плана при подтверждении.
- Раздел `Статистика`:
  - количество пользователей,
  - анкеты на проверке,
  - ожидают оплату,
  - оплачено,
  - отклонено,
  - активные подписки.
- Скрытая команда `/id` — показать Telegram ID админа.

### Супер-админ
- Меню `Настройки бота`:
  - `Настройка вывода` (редактирование пользовательских сообщений):
    - приветствие,
    - реквизиты оплаты,
    - подтверждение оплаты,
    - отказ по анкете,
    - сообщение после отправки анкеты,
    - шаблон успешного промокода,
    - шаблон ошибки промокода,
    - сообщение для пользователя без анкеты,
    - заголовок списка тарифов.
  - создание промокодов,
  - добавление тарифов,
  - просмотр списка тарифов,
  - управление администраторами (список, добавление, удаление).

## Архитектура и стек

- Python 3.12
- aiogram 3
- PostgreSQL (Docker)
- SQLAlchemy (async)
- Alembic (миграции)

Основные пути:
- `bot/main.py` — запуск приложения.
- `bot/src/user_components` — пользовательские роутеры и клавиатуры.
- `bot/src/admin_components` — админ-роутеры, модерация, настройки.
- `bot/src/db_components` — модели и менеджеры БД.
- `bot/src/FormManager` — управление вопросами анкеты в БД.
- `bot/alembic` — миграции БД.

## Запуск

1. Заполнить `bot/.env`:

```env
BOT_TOKEN=...
SUPER_ADMIN_ID=...
DB_HOST=postgres
DB_PORT=5432
DB_NAME=tg_bot_db
DB_USER=tg_bot_user
DB_PASSWORD=tg_bot_password
DB_ECHO=false
```

2. Поднять сервисы:

```bash
docker compose up -d --build
```

3. Проверить логи:

```bash
docker compose logs -f telegram-bot
```

## Полезные SQL-проверки

```bash
docker compose exec postgres psql -U tg_bot_user -d tg_bot_db -c "select id,status,user_id,personal_discount,promo_discount,promo_code_id from survey_submissions order by id desc limit 20;"
docker compose exec postgres psql -U tg_bot_user -d tg_bot_db -c "select position,text,question_type from survey_questions order by position;"
docker compose exec postgres psql -U tg_bot_user -d tg_bot_db -c "select id,name,duration_days,price,is_active from payment_plans order by id desc limit 20;"

```

## Ручной чек-лист тестирования

1. Пользователь: `/start` -> `Заполнить анкету` -> пройти анкету -> проверить `Статус профиля`.
2. Супер-админ: `/start` -> `Настройки бота` -> `Настройка вывода` -> изменить несколько сообщений.
3. Супер-админ: создать промокод, затем пользователем применить через `Промокод`.
4. Супер-админ: добавить тариф, затем пользователем открыть `Тарифы`.
5. Админ: `Модерация` -> одобрить анкету (без скидки и со скидкой).
6. Админ: `Ожидают оплату` -> подтвердить оплату -> выбрать тариф.
7. Админ: `Статистика` -> проверить изменение счетчиков.
8. Проверить `Назад` в админских разделах и скрытую команду `/id` у пользователя и админа.
